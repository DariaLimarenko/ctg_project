# CTG Image Processing — обработка КТГ-диаграммы на C++ и Python

Проект предназначен для обработки изображения медицинской диаграммы КТГ
(кардиотокографии) и преобразования графиков с изображения в числовые временные ряды.

Программа выполняет:

1. загрузку изображения КТГ;
2. коррекцию наклона изображения относительно линий сетки;
3. определение размерности изображения и длительности исследования;
4. выделение первого графика — **FHR**, частота сердечных сокращений плода;
5. выделение второго графика — **Toco**, токограмма / маточная активность;
6. построение временных рядов по двум графикам;
7. сохранение результата в CSV-файл;
8. построение итогового графика по полученным данным;
9. выполнение дополнительной постобработки:
   - создание отдельных CSV-файлов для каждого графика;
   - создание итогового Markdown-отчёта;
   - создание overlay-изображения с наложением найденных графиков на исходную диаграмму.

Проект реализован на двух языках:

- **C++** — основная обработка изображения с использованием OpenCV;
- **Python** — визуализация, сглаживание, анализ CSV-результата и постобработка.

## Назначение основных файлов

### Папка `data`

Содержит исходное изображение медицинской диаграммы:

```text

```

Файл должен быть настоящим изображением JPEG/PNG, а не пустым файлом.

Проверить файл в Linux/WSL можно командой:

Ожидаемый результат:

### Папка `output`

В эту папку сохраняются результаты работы программы.

| Файл | Назначение |
|---|---|
| `aligned.jpg` | изображение после коррекции небольшого угла наклона по линиям сетки |
| `roi_debug.jpg` | изображение с разделением на верхний и нижний график |
| `graph1_mask.jpg` | бинарная маска первого графика FHR |
| `graph2_mask.jpg` | бинарная маска второго графика Toco |
| `ctg_result.csv` | основной CSV-файл с двумя временными рядами |

| `toco_timeseries.csv` | отдельный CSV-файл второго графика Toco |

На нём найденные элементы первого графика отображаются красным цветом, а найденные элементы второго графика — синим цветом.

## Что делает C++ часть

C++ часть проекта отвечает за основную обработку изображения.

Используется библиотека **OpenCV**.

Основные этапы C++-части:

1. загрузка изображения;
2. проверка ориентации изображения;
3. поворот изображения, если оно вертикальное;
4. поиск линий сетки;
5. коррекция небольшого угла наклона изображения;
6. определение шага сетки;
7. расчёт длительности исследования;
8. разделение изображения на две области:
   - верхняя область — FHR;
   - нижняя область — Toco;
9. выделение тёмных пикселей графиков;
10. построение временных рядов;
11. сохранение данных в CSV;
12. сохранение промежуточных изображений для проверки.

Важно: текущая версия выполняет именно **коррекцию поворота** изображения по линиям сетки.
Она не выполняет полное перспективное выравнивание листа, как сканер документов.

---

## Что делает Python часть

Python часть проекта используется для визуализации, дополнительной обработки и формирования отчётных файлов.

Основные этапы Python-части:

1. чтение CSV-файла;
2. проверка наличия нужных колонок;
3. сглаживание временных рядов;
4. построение графика FHR;
5. построение графика Toco;
6. сохранение итогового изображения `ctg_plot.png`;
7. создание отдельных CSV-файлов:
   - `fhr_timeseries.csv`;
   - `toco_timeseries.csv`;
8. создание итогового отчёта `final_report.md`;
9. создание overlay-изображения `debug/combined_overlay.png`.

## Формат основного CSV-файла

Основной CSV сохраняется с разделителем `;`.

Файл:

```text
output/ctg_result.csv
```

Пример:

```csv
time_sec;time_min;FHR_bpm;Toco_value
0.000;0.000;150.0;47.9
1.000;0.017;150.1;50.8
2.000;0.033;150.2;47.5
3.000;0.050;150.2;46.9
```

Описание колонок:

| Колонка | Описание |
|---|---|
| `time_sec` | время исследования в секундах |
| `time_min` | время исследования в минутах |
| `FHR_bpm` | частота сердечных сокращений плода, уд/мин |
| `Toco_value` | значение токограммы |

## Описание алгоритма

### 1. Загрузка изображения

Программа получает путь к изображению:

```text
data/my_ctg.jpg
```

Изображение загружается с помощью OpenCV.

Если файл повреждён, отсутствует или имеет размер 0 байт, программа выдаст ошибку.

---

### 2. Коррекция наклона изображения

Исходное фото может быть немного наклонено.

Для коррекции наклона:

1. изображение переводится в оттенки серого;
2. выделяются границы;
3. с помощью преобразования Хафа находятся линии сетки;
4. вычисляется угол наклона;
5. изображение поворачивается на найденный угол.

Это нужно, чтобы координаты графиков можно было корректно переводить в числовые значения.

Результат сохраняется в файл:

```text
output/aligned.jpg
```

Файл `aligned.jpg` является изображением после коррекции небольшого угла наклона.
Он не является результатом полного перспективного выравнивания листа.

---

### 3. Определение размерности и длительности

Программа анализирует вертикальные линии сетки и оценивает расстояние между малыми клетками.

Так как на изображении указана скорость бумаги:

```text
1 см/мин
```

можно оценить длительность исследования по количеству клеток по горизонтали.

В результате программа определяет:

- ширину изображения в пикселях;
- высоту изображения в пикселях;
- шаг сетки;
- длительность исследования в секундах;
- длительность исследования в минутах.

---

### 4. Выделение двух графиков

Изображение разделяется на две области:

```text
Graph 1 — FHR
Graph 2 — Toco
```

Верхняя часть содержит график частоты сердечных сокращений плода.
Нижняя часть содержит токограмму.

Файл `roi_debug.jpg` показывает, как программа разделила изображение.

---

### 5. Построение масок графиков

Для каждой области программа строит бинарную маску:

```text
белые пиксели — найденная линия графика;
чёрные пиксели — фон и удалённые элементы.
```

Результаты сохраняются в файлы:

```text
graph1_mask.jpg
graph2_mask.jpg
```

Эти изображения используются для отладки и демонстрации промежуточного этапа обработки.

---

### 6. Извлечение временного ряда

Для каждого столбца изображения программа ищет тёмные пиксели, соответствующие линии графика.

Затем координата `y` переводится в числовое значение:

- для FHR — в уд/мин;
- для Toco — в условные значения токограммы.

После этого данные интерполируются и сохраняются как временной ряд.

---

### 7. Сохранение результата

Итоговые данные сохраняются в CSV-файл:

```text
output/ctg_result.csv
```

Затем Python строит график:

```text
output/ctg_plot.png
```

---

### 8. Postprocessing

После создания основного CSV выполняется дополнительная постобработка.

На этом этапе создаются:

```text
output/fhr_timeseries.csv
output/toco_timeseries.csv
output/final_report.md
output/debug/combined_overlay.png
```

Файл `final_report.md` содержит краткую статистику обработки:

- длительность исследования;
- количество строк в CSV;
- минимальное, максимальное и среднее значение FHR;
- минимальное, максимальное и среднее значение Toco;
- список созданных файлов;
- таблицу соответствия пунктам задания.

Файл `combined_overlay.png` показывает найденные элементы графиков поверх изображения диаграммы:

- красным цветом отображается найденный первый график FHR;
- синим цветом отображается найденный второй график Toco.

## Возможные проблемы и решения

### Ошибка: не удалось открыть изображение

Причины:

- файл отсутствует;
- неверный путь;
- файл повреждён;
- файл имеет размер 0 байт.

Проверка:

```bash
ls -lh data/my_ctg.jpg
file data/my_ctg.jpg
```

Если размер файла `0`, нужно заново скопировать изображение.

---

### Ошибка: `No module named pandas`

Нужно активировать виртуальное окружение и установить зависимости:

```bash
source .venv/bin/activate
pip install -r python/requirements.txt
```

---

### Ошибка: `python: command not found`

В некоторых системах команда `python` может отсутствовать.
Тогда можно использовать:

```bash
python3 python/visualize.py output/ctg_result.csv output/ctg_plot.png
```

или установить алиас:

```bash
sudo apt install python-is-python3
```

Если используется виртуальное окружение `.venv`, после активации обычно доступна команда `python`.

---

### Ошибка: `externally-managed-environment`

В новых версиях Ubuntu нельзя устанавливать Python-пакеты напрямую в системный Python.

Используйте виртуальное окружение:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r python/requirements.txt
```

---

### VS Code подчёркивает `#include <opencv2/opencv.hpp>`

Если проект собирается, но VS Code подчёркивает OpenCV красным, это проблема IntelliSense.

Создайте файл:

```text
.vscode/c_cpp_properties.json
```

С содержимым:

```json
{
  "configurations": [
    {
      "name": "WSL Ubuntu",
      "includePath": [
        "${workspaceFolder}/**",
        "/usr/include/opencv4"
      ],
      "compilerPath": "/usr/bin/g++",
      "cStandard": "c17",
      "cppStandard": "c++17",
      "intelliSenseMode": "linux-gcc-x64"
    }
  ],
  "version": 4
}
```

После этого в VS Code выполните:

```text
Ctrl + Shift + P
C/C++: Reset IntelliSense Database
```

## Ограничения проекта

Проект является учебным и демонстрационным.

Точность извлечения графиков зависит от качества изображения:

- наклона фотографии;
- освещения;
- контраста;
- наличия текста и лишних отметок;
- качества исходной печати;
- пересечения графика с сеткой.

Особенно это заметно для нижнего графика Toco, где рядом с кривой могут находиться подписи и служебные отметки. Они могут попадать в бинарную маску.

На overlay-изображении это может проявляться тем, что часть текста в нижней области также выделяется синим цветом.
Тем не менее основной ход второго графика сохраняется и используется для построения временного ряда.

8. добавить графический интерфейс;

## Важное медицинское замечание

Этот проект предназначен только для учебной обработки изображения и получения вспомогательных числовых данных.

Программа:

- не является медицинской диагностической системой;
- не ставит диагноз;
- не заменяет врача;
- не должна использоваться для самостоятельной интерпретации КТГ.

Полученные данные могут использоваться только как вспомогательная информация для анализа и демонстрации алгоритмов обработки изображений.

output/debug/combined_overlay.png
